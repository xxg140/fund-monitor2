# 整体行为准则（全程遵守）
 
1. 先思考再动手，不确定需求、需求不明确、用户表达模糊，信息不全时主动提问确认，绝不凭猜测直接写代码、乱改文件不瞎猜需求过度思考。（这条重中之重，任何时候都不得丢弃）

2. 简洁优先原则：用最少、最易懂的代码完成目标，拒绝冗余、复杂写法和过度设计，不做无关扩展。

3. 外科式精准修改：只改动必须修改的代码/内容，严格保留原有无关逻辑、结构、样式，不擅自重构全局。

4. 目标驱动执行：先明确任务成功标准，再动手开发；完成后必须验证功能可用，不能只写完不测试。

5. 确定性任务代码化：格式转换、批量处理、重复执行等固定流程，优先写工具函数/代码实现，不依赖模型反复尝试。

6. 长任务控Token：复杂、多步骤任务先总结当前状态，精简上下文，避免无效对话浪费token。

7. 方案单一不混乱：遇到方案冲突时选定最优一种并说明理由，禁止多方案混用、左右摇摆。

8. 先读现有文件再写：修改前先阅读项目已有代码、样式、逻辑，优先复用现有功能，不重复造轮子。

9. 业务级有效测试：测试重点验证业务逻辑正确性，不只检查接口有无返回、页面是否显示。

10. 长任务设置检查点：复杂任务分步执行、每步自查状态；逻辑混乱、状态说不清时立即暂停，先梳理再继续。

11. 跟随项目原有习惯：严格沿用项目现有的代码风格、命名、结构、写法，不私自改规范、加新风格。

12. 诚实反馈不隐瞒：代码报错、逻辑失败如实说明；不确定、不会做直接标注「不确定」，不编造、不硬写错误内容。

13. 中文命名规范（HTML/CSS/JS通用）代码在不会因为文字出问题的情况下，优先使用中文命名
所有类名、标识、变量、注释优先使用中文，保持可读性统一

14. 所有样式统一放在「样式.css」，不分散写在其他文件

15. 每次修改代码后更新页面顶部右上角的版本号`APP_VERSION`，这样用户第一时间能识别出来有没有因为浏览器缓存照成文件修改未同步

16. 任何时候临时文件都不准放在，项目文件里，直接放在桌面醒目的地方，方便用户删除，或者自己用完第一时间删除

17. 如用户要求修改此文件，以上内容任何时候都不准修改，只能修改以下内容


# 基金持仓监控项目说明

## 开发规范

## CSS 变量命名

- **全部使用中文**命名，不要用英文变量名
- `:root` 定义浅色主题，`body.dark` 定义深色主题（只覆盖颜色值，变量名保持一致）
- 当前变量结构：
  - `--页面背景` `--卡片背景` `--次级区域背景` `--三级区域背景`：背景层级
  - `--主文字色` `--次文字色` `--弱化文字色`：文字层级
  - `--默认边框` `--浅色边框` `--悬停边框`：边框层级
  - `--强调色` `--强调深色` `--强调浅色`：强调色系
  - `--上涨色` `--上涨背景` `--下跌色` `--下跌背景`：涨跌颜色

## 五类按钮体系

| 类名 | 用途 | 背景 |
|------|------|------|
| `btn-main` | 主页按钮 | `var(--卡片背景)` |
| `btn-tab` | 弹窗切换按钮 | `transparent` |
| `btn-action` | 弹窗操作按钮 | `transparent` |
| `btn-danger` | 危险按钮 | `var(--危险按钮背景)` |
| `btn-icon` | 图标按钮 | `none` |

- 按钮颜色统一由对应的 CSS 变量控制，不要单独改某个按钮的颜色
- 危险按钮的 `--危险按钮背景` 和 `--危险按钮边框` 只定义在 `:root`，深色模式不变（保持醒目）

## CSS 类名规范

- **能用中文的地方用中文**，类名简洁清晰
- 已有中文工具类：`.左对齐` `.居中对齐` `.右对齐` `.弹性1` `.可点击` `.字号11` `.字号12` 等
- 不要写内联 `style="..."`，优先用已有的 CSS 类，没有的再新增

## 按钮垂直居中

- btn-main / btn-action / btn-icon：使用 `display: flex; align-items: center; justify-content: center;`
- btn-tab / btn-danger：使用 `display: inline-block; line-height: 2;`（内联文本排列时对齐更好）
- 修改按钮时注意不要破坏垂直对齐

## 项目定位

这是一个本地单页 Web 应用，用于监控基金持仓、实时估值、指数行情、概念板块和行业板块。项目没有后端服务，主要通过浏览器直接打开 `index.html` 使用，个人本地使用优先。

核心特点：

- 纯静态页面：`index.html + 样式.css + 工具函数.js + 主逻辑.js`
- 数据保存在浏览器 `localStorage`
- 外部行情和基金数据来自东方财富、腾讯行情等公开接口
- 图表依赖 CDN 加载的 Chart.js
- 页面、样式变量、工具类和说明文档尽量使用中文

## 启动方式

- 入口文件：`index.html`
- 使用方式：直接用浏览器打开 `index.html`
- 注意：趋势图、行情、净值、板块等功能需要联网
- 当前版本号：`工具函数.js` 里的 `APP_VERSION`
- 版本显示位置：页面顶部标题“基金持仓监控”旁边的 `versionTag`，不是左下角

## 主要功能

- 基金导入：支持单个导入和批量导入。
- 持仓监控：展示基金名称、代码、日期/净值、实时估值、今日收益、持有收益等。
- 汇总栏：展示总持仓、实时总估值、今日收益、持有总收益。
- 分组管理：支持新建、重命名、删除分组，基金可移动到不同分组。
- 交易记录：支持加仓、减仓、待确认、已确认、已撤回状态。
- 基金详情：趋势图、前 10 持仓、历史净值、修改持仓、删除基金。
- 指数行情：支持指数卡片、指数选择和指数详情图表。
- 板块行情：支持概念板块和行业板块，并可按名称、涨幅、资金流入排序。
- 主题：支持浅色、深色、跟随系统。
- 隐私：支持隐藏金额。
- 列设置：支持配置基金列表列显示。

## 核心数据流

1. `index.html` 加载页面 DOM、`样式.css`、Chart.js、`工具函数.js` 和 `主逻辑.js`。
2. `工具函数.js` 提供全局常量、格式化、安全读取和安全转义。
3. `主逻辑.js` 启动时读取 `localStorage` 中的基金、分组、指数、列设置、主题等配置。
4. `refreshAll()` 拉取基金、指数和待确认交易所需数据。
5. `renderFunds()` 根据当前分组、排序、列设置和基金数据渲染列表与汇总栏。
6. 用户执行导入、交易、修改、删除、分组、板块关联等操作后，通过 `saveFunds()` 或 `safeSet()` 写回本地存储。

## 关键函数索引

`工具函数.js`：

- `APP_VERSION`：页面顶部标题旁边显示的版本号。
- `safeGet()` / `safeSet()`：安全读取和写入 `localStorage`。
- `escapeHtml()` / `safeHtml()`：转义用户输入，避免 HTML 注入。
- `formatMoney()` / `formatDate()`：金额和日期格式化。

`主逻辑.js`：

- `loadStorage()`：读取本地基金和指数配置。
- `saveFunds()`：保存基金持仓主数据。
- `createFundObj()`：创建基金对象。
- `calcSharesFromPurchaseDate()`：根据买入日期和净值计算份额。
- `fetchFundData()`：获取基金基础/实时数据。
- `fetchFundHistory()`：获取历史净值。
- `fetchFundHoldings()`：获取基金前 10 持仓。
- `fetchIndices()`：获取指数行情。
- `calcTodayProfit()`：计算今日收益。
- `calcMarketValue()`：计算实时估值。
- `renderFunds()`：渲染基金列表和汇总栏。
- `openFundDetailModal()`：打开基金详情弹窗。
- `refreshAll()`：刷新全部基金和指数。
- `submitTradeRaw()`：提交加仓/减仓。
- `loadSectorData()`：加载概念/行业板块数据。

## 补充开发约定

### 中文优先

- 能用中文的地方尽量用中文，包括文档、注释、CSS 变量和工具类。
- 已有英文类名和函数名可以保留，不要为了中文化做大规模无意义重命名。
- 新增说明文档优先中文。

### CSS 类名

- 能复用已有类就复用，避免重复造一堆相近类。
- 已有中文工具类示例：`.左对齐`、`.居中对齐`、`.右对齐`、`.弹性1`、`.可点击`、`.字号11`、`.字号12`。
- 新增样式优先写到 `样式.css`，减少新的内联 `style="..."`。
- 现有 HTML 中已有一些内联样式，不要求一次性清理，避免无关大改。

### 安全渲染

- 用户输入、基金名称、板块名、分组名等内容进入 `innerHTML` 前必须转义。
- 优先使用 `safeHtml()` 或 `escapeHtml()`。
- 不要把未清洗的字符串直接拼进 HTML。

## 重要业务规则

- 今日收益、持有收益、实时估值的计算逻辑比较敏感，核心在 `calcTodayProfit()`、`calcDisplayProfit()`、`calcMarketValue()`。
- 买入日期会影响份额和成本计算，QDII 基金存在日期补偿逻辑。
- 加仓/减仓区分 3 点前和 3 点后。
- 净值未更新时，交易会进入待确认队列，后续由 `checkPendingOperations()` 尝试确认。
- 减仓要扣除待确认减仓后的可用份额。
- 历史净值缓存有过期时间和数量上限，避免频繁请求和本地存储过大。
- 前 10 持仓缓存时间较长，因为基金持仓通常按季度更新。

## 高风险修改点

- 不要随意更改 `fund_monitor_v3` 的数据结构；如果必须改，要写兼容迁移。
- 不要删除交易状态字段，待确认、已确认、已撤回会影响持仓计算。
- 不要直接重命名 CSS 变量，否则浅色/深色主题可能一起坏。
- 不要把 JSONP 接口直接改成普通 `fetch` 后就默认可用。
- 不要只改页面渲染而忘记保存逻辑。
- 不要忽略移动端，项目有触摸、拖拽、弹窗和图表交互。
- 不要为小需求引入大型框架或构建系统，除非用户明确要求。

## 修改建议

接手修改时建议按这个顺序看：

1. 先读 `工具函数.js`，了解全局常量、安全函数和版本号。
2. 再读 `主逻辑.js` 的数据区、存储区、刷新区和渲染区。
3. 改样式前先看 `样式.css` 顶部变量和按钮体系。
4. 改弹窗前先看 `index.html` 中对应 DOM 结构。
5. 改完至少检查页面能打开、控制台无明显报错、弹窗能打开、主题能切换。

## 可考虑的后续优化
