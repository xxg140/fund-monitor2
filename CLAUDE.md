## 项目概述

纯静态单页 Web 应用，用于监控基金持仓、实时估值、指数行情、概念板块和行业板块。无后端服务，直接浏览器打开 `index.html` 使用。数据保存在 `localStorage`，外部数据来自东方财富、腾讯行情等公开接口。图表依赖 CDN Chart.js。

## 文件结构

```text
基金/
├─ index.html              # 页面结构、弹窗 DOM、APP_VERSION 定义、脚本/样式引入
├─ js+css/
│  ├─ 样式.css             # 所有样式、CSS 变量（中文命名）、主题、按钮体系、响应式
│  ├─ 工具函数.js           # 全局常量、格式化函数、安全读写 localStorage、HTML 转义
│  ├─ 主逻辑.js            # 核心业务逻辑（160KB+）：数据、存储、刷新、渲染、交易、图表、分组、板块
│  └─ 指数图基金图.js       # 指数/基金图表绘制（Chart.js 封装）
└─ CLAUDE.md
```

## 启动方式

直接浏览器打开 `index.html`。趋势图、行情、净值、板块等功能需要联网。

## 版本号

- `APP_VERSION` 在 `index.html` 第 7 行全局 `<script>` 中定义（如 `var APP_VERSION = 'V1'`）
- 版本显示在页面顶部标题"基金持仓监控"旁边的 `versionTag` 元素
- CSS/JS 引用通过 `APP_VERSION` 参数实现缓存破除

能用中文的地方用中文。已有中文工具类：`.左对齐` `.居中对齐` `.右对齐` `.弹性1` `.可点击` `.字号11` `.字号12` 等。不要写内联 `style="..."`，优先用已有 CSS 类。

## 安全渲染

用户输入、基金名称、板块名、分组名等内容进入 `innerHTML` 前必须转义。使用 `safeHtml()` 或 `escapeHtml()`（定义在 `工具函数.js`）。

## 核心数据流

1. `index.html` 加载 DOM、`样式.css`、Chart.js CDN、`工具函数.js`、`指数图基金图.js`、`主逻辑.js`。
2. `工具函数.js` 提供全局常量、格式化、安全读写。
3. `主逻辑.js` 启动时读取 `localStorage` 中的基金、分组、指数、列设置、主题等配置。
4. `refreshAll()` 拉取基金、指数和待确认交易所需数据。
5. `renderFunds()` 根据当前分组、排序、列设置和基金数据渲染列表与汇总栏。
6. 用户操作后通过 `saveFunds()` 或 `safeSet()` 写回本地存储。

## 外部数据来源

项目依赖多个第三方接口，很多接口使用 JSONP 或脚本注入，**不能随意改成普通 fetch**。

- Chart.js CDN：`cdn.jsdelivr.net/npm/chart.js@4.4.0` + `chartjs-adapter-date-fns@3`
- 东方财富基金数据：`fundmobapi.eastmoney.com`、`fund.eastmoney.com`、`fundf10.eastmoney.com`
- 腾讯行情：`qt.gtimg.cn`、`ifzq.gtimg.cn`、`web.ifzq.gtimg.cn`
- 东方财富板块：`push2delay.eastmoney.com`

## localStorage 重要键

修改数据结构时必须考虑旧数据兼容，不要直接破坏用户已有持仓。

- `fund_monitor_v3`：基金持仓主数据
- `fund_monitor_indices`：用户选择的指数列表
- `fund_monitor_groups`：分组列表
- `fund_monitor_active_group`：当前选中的分组
- `fund_monitor_theme`：主题模式（`auto` / `dark` / `light`）
- `fund_monitor_auto_refresh`：是否自动刷新
- `fund_monitor_hide_mv`：是否隐藏金额
- `fund_monitor_columns`：列显示配置
- `fund_monitor_col_version`：列配置版本
- `fund_monitor_tradetime`：每只基金的交易时间偏好
- `fund_history_cache_v2`：历史净值缓存
- `fund_holdings_cache_v2`：前 10 持仓缓存

## 重要业务规则

- 今日收益、持有收益、实时估值的计算逻辑敏感，核心在 `calcTodayProfit()`、`calcDisplayProfit()`、`calcMarketValue()`。
- 买入日期影响份额和成本计算，QDII 基金存在日期补偿逻辑。
- 加仓/减仓区分 3 点前和 3 点后。
- 净值未更新时，交易进入待确认队列，后续由 `checkPendingOperations()` 尝试确认。
- 减仓要扣除待确认减仓后的可用份额。

## 高风险修改点

- 不要随意更改 `fund_monitor_v3` 数据结构；如必须改，要写兼容迁移。
- 不要删除交易状态字段（待确认、已确认、已撤回会影响持仓计算）。
- 不要直接重命名 CSS 变量（浅色/深色主题可能一起坏）。
- 不要把 JSONP 接口直接改成普通 fetch。
- 不要只改渲染而忘记保存逻辑。
- 不要忽略移动端（项目有触摸、拖拽、弹窗和图表交互）。
- 不要为小需求引入大型框架或构建系统。

## 修改建议

接手修改时按顺序看：

1. 先读 `js+css/工具函数.js`，了解全局常量、安全函数。
2. 再读 `js+css/主逻辑.js` 的数据区、存储区、刷新区和渲染区。
3. 改样式前先看 `js+css/样式.css` 顶部变量和按钮体系。
4. 改弹窗前先看 `index.html` 中对应 DOM 结构。
5. 改完至少检查页面能打开、控制台无明显报错、弹窗能打开、主题能切换。
